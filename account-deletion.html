타닥타닥 계정 삭제 및 데이터 삭제 안내

Published: 2026-03-06 Independent URL

Quick Summary

  • 앱 내 `프로필 > 계정 관리 > 삭제하기` 경로에서 계정 삭제를 직접 진행할 수 있습니다.
  • 삭제가 확정되면 프로필, 기록, 작성 글, 좋아요/가져오기, 게시글 신고 이력 등 계정 귀속 데이터가 삭제 대상으로 처리됩니다.
  • tracking/피드백/광고/ATT/review/user report 로그는 삭제가 아니라 계정 식별자를 제거한 뒤 운영 목적 범위에서 보관될 수 있습니다.

1. 삭제 요청 방법

  1. 타닥타닥 앱 실행
  2. `프로필 > 계정 관리` 이동
  3. `삭제하기` 선택
  4. `삭제 안내 확인` 후 최종 `계정 삭제` 확인

앱 접근이 어려운 경우에는 omilsoft@gmail.com 으로 삭제 지원을 요청할 수 있습니다.

2. 삭제 대상 데이터

  • 프로필 정보, 닉네임, 설정값
  • 연습 기록, 랭크 관련 데이터
  • 개인 작성 글과 모두의 공간 공개 글
  • 좋아요, 가져오기, 게시글 신고 이력
  • 작성자 숨김/차단 목록
  • legacy migration audit 등 계정에 직접 귀속된 일부 감사 기록

3. 삭제 후 예외 보관 또는 익명화 가능 데이터

  • tracking_events는 user_id, anonymous_id, session_id 같은 계정 식별자를 제거하고 payload 내 식별 필드를 마스킹한 뒤 운영 로그로 남을 수 있습니다.
  • 앱 내 피드백/버그 제보는 user_id를 제거하고 message를 삭제 계정용 문구로 치환한 뒤 metadata 식별 필드를 정리한 상태로 남을 수 있습니다.
  • ad_interstitial_events, att_prompt_events, review_funnel_events는 user_id 및 metadata 식별 필드를 제거한 뒤 운영 목적 범위에서 남을 수 있습니다.
  • 사용자 신고(community_user_reports)는 신고자/대상자 식별자를 제거하고 target_snapshot의 식별 필드를 마스킹한 뒤 운영 검토 기록으로 남을 수 있습니다.
  • Apple App Store 또는 Google Play의 구매/복원/환불 이력은 각 스토어 사업자가 별도로 보관합니다.
  • Google, Apple, Kakao 계정 자체의 해지 또는 삭제는 각 로그인 제공자 정책을 따르며, 타닥타닥 계정 삭제로 자동 처리되지 않습니다.

4. 게스트 계정 삭제

  • 게스트 계정을 삭제하면 해당 게스트 상태와 그 계정에 귀속된 작성 글/기록이 함께 제거됩니다.
  • 게스트 계정은 모두의 공간에 공개 공유할 수 없으며, 공개 공유에는 계정 연동이 필요합니다.

5. 처리 방식과 유의사항

  • 앱 내 최종 확인이 끝나면 계정 삭제는 즉시 실행되며 복구할 수 없습니다.
  • 이메일 문의는 영업일 기준 순차적으로 확인하고 회신합니다.
  • 본인 확인 또는 삭제 범위 확인을 위해 추가 정보를 요청할 수 있습니다.
